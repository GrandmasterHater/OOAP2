  Принципы перечисленных видов наследования понятны:
  - При наследовании вариаций и функциональной вариацией наследник переопределяет поведение метода, но сигнатура остаётся без изменений.
    При наследовании с вариацией типа изменяется сигнатура метода (появляется перегрузка). Общая же рекомендация в целом не переопределять методы,
    поскольку это может приводить к непредсказуемому поведению переопределяемых методов. Лучше использовать чистую версию композиции.
  - Наследование с конкретизацией - схема при которой реализация либо полностью отложена, либо частично присутсвует в родительском классе и
    должна быть дополнена в наледниках.
  - Структурное наследование - добавляет полноценную абстракцию, которая относится к добавляемому типу не как отношение IS-A, а как операция над всем типом.

Примеры:
  - Наследование вариаций в примере Java понятно - газовая плита с электроподжигом является разновидностью газовой плиты с дополнением функционала
    автоматическим поджигом. При этом сигнатура метода в наследнике меняется, таким образом это наследование с вариацией типа.
  - Также пример на Java отдельно показывает наследование с функциональной вариацией на примере HttpsRequest где родительский тип дополняется возможностью
    шифрования сообщения.
  - Java пример с структурным наследованием также понятен - тип Summable определяет общий набор операций над всем, что поддерживает
    суммирование (публичный контракт). Тип MyInt реализует этот контракт. При этом операции связанные с контрактом работают над типом MyInt, а не
    наследуют свойства типа Summable.
  - В примере на Python структурное наследование и наследование с функциональной вариацией полностью понятны. Наследование с вариацией типа и
    наследование с конкретизацией сами по себе также понятны, но их совместное применение в типе FileAPI вызывает вопрос о том насколько допустимо применять
    в рамках одного типа сразу несколько видов наследования. Полагаю, что на практике лучше придерживаться одного вида наследования, чтобы
    избежать дальнейшей путаницы и сложностей при проектировании.
