   Оценка примеров даётся тяжело т.к. пока еще нет увернного понимания ковариантности в контексте сигма исчисления. Из того, что понял:
   - У Python есть приближенная возможность делать ковариантные вызовы, однако она не совсем та, что описана в сигма-исчислении, поскольку
     в данном случае ковариантен только тип параметр конструктора Box. В остальном картина ближе к классичечкому полиморфизму где есть некоторый
     обобщенный тип и реализация метода зависит от этого полиморфного типа. 
   - В Java также видно ковариантность типа, а не выбор реализации метода согласно указанному типу.
