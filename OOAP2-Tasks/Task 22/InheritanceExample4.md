   В качестве примера наследования вида можно привести иерархию оружия, где первый способ классификации это качество оружия (обычное, редкое, реликтовое, легендарное), а второй, например, 
вид (меч, лук, револвер, гранатомёт и т.д.). Тут присутсвует несколько способов классификации, каждый из которых важен для общей картины игры и независим. Есть возможность комбинирования 
критериев, каждый вид оружия влияет на механику игры, поэтому АТД должен быть хорошо продуман. В итоге тип Weapon будет связан связью HAS-A с типами Quality и WeaponType:
```
public class Weapon
{
    Quality;
    WeaponType;
}
```
