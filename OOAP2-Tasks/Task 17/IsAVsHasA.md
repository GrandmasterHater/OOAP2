### Пример 1

   Например есть игра с набором разных персонажей. В игровом матче есть роли в команде - атакующий, целитель, танк. Каждый персонаж может 
принимать разную роль в зависимости от матча. На первый взгляд связь между персонажем и ролью может показаться "является" - персонаж является 
воином/целителем/защитником:

```
Warrior : Character

Healer : Character

Tank : Character
```

Но, поскольку роль от матча к матчу может меняться, то уместнее будет определить полиморфный тип Role который будет описывать поведение персонажа. 
Тогда связь между ролью и персонажем смениться на "содержит":
```
Character
{
   private Role;
}
```

### Пример 2

   Представим команду разработки в которой есть ПМ, QA, фронт и бекенд разработчики. Если ежедневный мит команды каждый раз фасилитирует один человек, 
к примеру ПМ, то можно выявить связь между должностью "ПМ" и ролью "фасилитатор" как "является" - ПМ является фасилитатором. Но если обязанность фасилитатора
ежедневно занимает каждый из участников, то между типом "должность" и типом "роль" возникает связь "содержит" - разработчик/QA содержит роль "фасилитатор". 
Сама роль может быть также представлена типом Role:

```
TeamMember
{
    private Role; // например, Facilitator или Speaker.
}
```
